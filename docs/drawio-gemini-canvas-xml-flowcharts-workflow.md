# Generate draw.io flowcharts with Gemini Canvas XML

This note covers an external workflow that connects Gemini Canvas with draw.io / diagrams.net.
It is not a Google Workspace product page, but it is useful when using Gemini to generate diagrams for draw.io.

## Core idea

Ask Gemini Canvas to generate plain, uncompressed draw.io XML for a flowchart, then paste or import that XML into draw.io.

draw.io documents that AI systems can generate native draw.io XML in either of these forms:

1. Full `.drawio` XML:

```xml
<mxfile>
  <diagram id="page-1" name="Page-1">
    <mxGraphModel>
      <root>
        <mxCell id="0" />
        <mxCell id="1" parent="0" />
      </root>
    </mxGraphModel>
  </diagram>
</mxfile>
```

2. Simplified `mxGraphModel` only:

```xml
<mxGraphModel>
  <root>
    <mxCell id="0" />
    <mxCell id="1" parent="0" />
  </root>
</mxGraphModel>
```

The simplified `mxGraphModel` format is usually easier for Gemini to generate and is accepted by draw.io for simple single-page diagrams.

## Recommended prompt for Gemini Canvas

```text
Generate a draw.io / diagrams.net flowchart as plain uncompressed XML.
Return only a valid <mxGraphModel> XML fragment, no Markdown fences, no explanation.

Requirements:
- Include the mandatory root cells:
  <mxCell id="0" />
  <mxCell id="1" parent="0" />
- Use unique IDs.
- Use vertex="1" for shapes and edge="1" for connectors.
- Put all user-visible text in mxCell value attributes.
- Use rounded rectangles for process steps and rhombus shapes for decisions.
- Use orthogonal connectors with arrows.
- Do not generate compressed or base64 XML.

Flowchart to generate:
[describe the process here]
```

## Import into draw.io

Documented method:

1. Open draw.io / diagrams.net.
2. Select `Extras > Edit Diagram`.
3. Paste the XML generated by Gemini.
4. Choose one of:
   - Replace existing drawing
   - Open in new window
   - Add to existing drawing
5. Click OK.

Alternative AI clipboard method in draw.io:

1. Open the Generate / AI dialog in draw.io.
2. Use clipboard actions such as:
   - Copy diagram to clipboard
   - Copy selection to clipboard
   - Update diagram from clipboard
   - Insert diagram from clipboard
3. Paste the XML into Gemini, ask for edits, copy the modified XML back, then update or insert from clipboard.

## About Ctrl+Shift+V

A practical shortcut often used in this workflow is Ctrl+Shift+V / Cmd+Shift+V to paste raw text without formatting.
This is useful when moving XML from Gemini into draw.io.

However, the official draw.io documentation for XML import points to `Extras > Edit Diagram` and the AI clipboard actions, not to Ctrl+Shift+V as the primary import mechanism. If the shortcut fails or pastes formatted text, use `Extras > Edit Diagram` instead.

## Validation checklist

Before pasting into draw.io, verify that Gemini output:

- Starts with `<mxGraphModel` or `<mxfile`.
- Contains `<root>`.
- Contains the mandatory cells `id="0"` and `id="1" parent="0"`.
- Uses unique IDs.
- Uses `vertex="1"` for shapes.
- Uses `edge="1"` for connectors.
- Is plain XML, not Markdown, not escaped HTML, not base64.

## Minimal example

```xml
<mxGraphModel dx="1000" dy="700" grid="1" gridSize="10" guides="1" tooltips="1" connect="1" arrows="1" fold="1" page="1" pageScale="1" pageWidth="850" pageHeight="1100" math="0" shadow="0">
  <root>
    <mxCell id="0" />
    <mxCell id="1" parent="0" />
    <mxCell id="start" value="Start" style="ellipse;whiteSpace=wrap;html=1;fillColor=#d5e8d4;strokeColor=#82b366;" vertex="1" parent="1">
      <mxGeometry x="120" y="80" width="120" height="60" as="geometry" />
    </mxCell>
    <mxCell id="step1" value="Do the task" style="rounded=1;whiteSpace=wrap;html=1;fillColor=#dae8fc;strokeColor=#6c8ebf;" vertex="1" parent="1">
      <mxGeometry x="120" y="180" width="120" height="60" as="geometry" />
    </mxCell>
    <mxCell id="edge1" value="" style="endArrow=block;html=1;rounded=0;orthogonalLoop=1;jettySize=auto;" edge="1" parent="1" source="start" target="step1">
      <mxGeometry relative="1" as="geometry" />
    </mxCell>
  </root>
</mxGraphModel>
```
